为确保项目信息及时且恰当地规划、收集、生成、发布、存储、检索、管理、控制、监督和最终处置所需。

大多数项目失败的重要原因是沟通失败！！！信息技术的不断变化，使得计算机专业人士与非专业人士的沟通越变负责，项目经理需要话费大量无规律的时间与相关干系人进行沟通，以确保项目在正确的轨道上如期保质保量的完成。
## 沟通模型
- 编码
  把思想或想法转化为他人能理解的语言。
- 信息和反馈信息
  编码过程所得到的的结果。
- 媒介
  传递信息的方法。
- 噪声
  干扰信息传输和理解的一切因素(如距离、新技术、缺乏背景信息等)。
- 解码
  把信息还原成有意义的思想或想法。
沟通5个基本状态：`已发送`、`已收到`、`以理解`、`已认可`、`已转化积极的行动`。

沟通渠道：
- 个人型
  - 个人沟通渠道
    两个或更多的人直接相互交流，如：面对面、电话、邮件。
    口头传播影响，在许多产品领域行之有效。
  - 非个人沟通渠道
    - 主要媒体
      - 报刊媒体、广播媒体、展示媒体。
    - 氛围
      特别设计的环境，建立并加强买主购买某一产品的倾向。 
    - 活动
      安排好的事件，项目标受众传达信息。
- 非个人型
  组织中的沟通渠道：
  - 正式沟通渠道
    在组织系统内，依据一定的组织原则所进行的信息传递与交流，如：`传达文件`、`召开会议`、`上下级之间的定期情报交换`、`参观访问`、`技术交流`、`市场调查`等。

    优点：沟通效果好，比较严肃，约束力强，易于保密，使信息沟通保持权威性。
    缺点：由于依靠组织系统层层的传递，所以较刻板，沟通速度慢。
  - 非正式沟通渠道
    除正式沟通以外的的信息交流和传递以及相互之间的回馈，已达成双方利益和目的的一种方式，不受组织简单，自由选择沟通渠道。如：成员`私下交流看法`、`朋友聚会`、`传播谣言和小道消息`等。
    非正式沟通时正式沟通的有机补充。

    优点：沟通形式不拘，直接明了，速度快，容易及时了解到正式沟通难以提供的“内幕新闻”。
    缺点：难以控制，传递的信息不确切，易于失真、曲解，而且可能导致小集团、小圈子，影响人心稳定和团体的凝聚力。

    特点：
    - 消息月新鲜，人们谈论得越多
    - 对人们工作有影响的，最容易招致人们谈论
    - 最为人们所熟悉的，更容易让人们谈论
    - 在工作中有关系的人，更容易牵扯到同一传闻中
    - 在工作上接触多的人，更容易牵扯到同一传闻中

    应充分注意，以杜绝起消极作用的“小道消息”。
需要注意`沟通技巧`：
- 赞美对方
- 移情入境
- 轻松幽默
- 袒胸露怀
- 求同存异
- 深入浅出
- 聆听中的技巧
- 使用目光接触和对视
- 展现赞许性的表示
- 避免分心的举动或手势
- 适时合理地体温
- 正确有效地复述
- 避免随便打断对方
- 尽量做的多听少说
- 使听者与说者的角色顺利转换

沟通中的表述技巧：
- 预先准备思路和提纲
- 及时调整和修订编码
- 及时合理地征询意见
- 避免过度表现自己
- 尽量言简意赅

## 1.规划沟通管理
根须干系人的信息需要和要求及组织的可用资产情况，识别和记录与干系人的最有效率且最有效果的沟通方式，制订合适的项目沟通方式和计划。
::: tip
规划沟通对项目最终成果非常重要，沟通规范不当，可能导致各种问题。正确的形式在正确的时间把信息提供给正确的受众，有利于项目的促成。
:::
主要考虑：
- 信息应该存储在什么地方
- 以什么形式存储信息
- 如何检索信息
- 信息需求方和信息的权限
- 信息所被需要的时间
- 是否需要考虑时差、语言障碍和跨文化因素等

- [项目管理计划](01.整体管理.md#项目管理计划)
  
- [干系人登记册](08.干系人管理.md#干系人登记册)
  
- [事业环境因素](01.整体管理.md#事业环境因素)和[组织过程资产](01.整体管理.md#组织过程资产)
### 技术与工具
- [会议](01.整体管理.md#会议)
- #### 沟通需求分析
  通过沟通需求分析，确定项目干系人的信息需求，包括所需信息的类型和格式，以及信息对干系人的价值。
  潜在沟通渠道总量为： $n(n-1)/2$  n代表干系人的数量。

  常用于识别和确定项目沟通需求的信息包括（但不限于）：
  - OBS 组织结构图
  - 项目组织与干系人之间的责任关系
  - 项目所涉及的学科、部门和专业
  - 参与项目的人数与项目地点
  - 内部信息需求
  - 外部信息需求
  - 来自干系人登记册的干系人信息和沟通需求
- #### 沟通技术
  可以采用各种技术在项目干系人之间传递信息，可能影响沟通技术选择的因素包括：
  - 信息需求的紧迫性
  - 技术的可用性
  - 易用性
  - 项目环境
  - 信息的敏感性和保密性
- #### 沟通方法
  - 交互式沟通
    在两方或多方之间进行多向信息交换。确保全体参与者对特定话题达成共识的最有效方法。例如：`会议`、`电话`、`即时通信`、`视频会议`等。
  - 推式沟通
    把消息发送给需要接收这些消息的特定接收方。可以确保信息的发生，但不能确保信息送达受众或被目标受众理解。例如：`信件`、`备忘录`、`报告`、`电子邮件`、`传真`、`语音邮件`、`日志`、`新闻稿`等。
  - 拉式沟通
    用于信息量很大或受众很多的情况，要求接受者自主自行地访问信息内容，。例如：`企业内网`、`电子在线课程`、`经验教训数据看`、`知识库`等。
### 输出
- #### 沟通管理计划
  描述如何对项目沟通进行规划、结构化和监控。
  计划内容包括：
  - 通用术语表
  - 干系人的沟通需求
  - 需要沟通的信息，包括：语言、格式、内容、详细程度等
  - 发布信息的原因
  - 发布信息机告知收悉或做出回应的时限和频率
  - 负责沟通相关信息的人员
  - 负责授权保密信息发布的人员
  - 将要接收信息的个人或小组
  - 传递信息的技术或方法
  - 为沟通活动分配的资源，包括：时间和预算
  - 问题升级程序，用于规定下层员工无法解决问题时的上报时限和上班路径
  - 随项目进展，对沟通管理计划进行更新与优化的方法
  - 项目信息流向图、工作流程、报告清单、会议计划等
  - 沟通制约因素
  - 项目状态会议、项目团队会议、网络会议、电子邮件信息等的指南和模板
- 项目文件更新
  如：项目进度计划、干系人登记册等。
## 2.管理沟通
::: tip
根据沟通管理计划，生成、收集、分发、存储、检索及最终处置项目信息，促进项目干系人之间实现有效率且有效果的沟通。
:::
不仅仅是将相关信息发布出去，还要设法确保信息被正确地生成、接受和理解，并为干系人获取更多信息、展开澄清和讨论创造机会。
::: note
#### 沟通四种方式
| 沟通方式 | 掌握信息的能力 | 控制强度 | 参与程度 | 是否挺起他人意见和想法 |
| :------: | :------------: | :------: | :------: | :--------------------: |
|   讨论   |       1        |    1     |    4     |           是           |
|   征询   |       2        |    2     |    3     |           是           |
|   推销   |       3        |    3     |    2     |           否           |
|   叙述   |       4        |    4     |    1     |           否           |
:::

## 3.控制沟通
《PNBOK 第六版》改为`监督沟通`。
::: note
在项目的整个生命周期中，对沟通进行监督和控制，确保满足项目干系人对信息的需求，随时确保所有沟通参与者之间的信息流动的最优化。并评估和控制项目沟通的影响和影响的反应，以确保在正确的时间把正确的信息传递给正确的受众。
:::
::: danger
`管理沟通`是要开展有效率和有效果的沟通；
`控制沟通`是为了检查沟通的效率和效果，发现问题需要提出变更请求。
二者属于不同过程组。
:::

- 问题日志
- 工作绩效数据
- [组织过程资产](01.整体管理.md#组织过程资产)
