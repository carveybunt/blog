质量管理 Quality Management原创

国际标准化组织（ISO）对质量（quality）的定义：反映实体满足主体明确和隐含需求的能力的特性总和。
GB/T19000-2008 对质量的定义为：一组固有特性满足要求的程度。

规划质量管理、实施质量保证、控制质量。
确定了质量工作政策、目标与职责的各过程和活动，从而使项目满足其预定的需求。
质量管理是指确定质量方针、目标和职责，并通过质量体系中的质量规划、质量保证和质量控制以及质量改进来使其实现所有管理质量的全部活动。
【质量管理 白嫖](https://ke.qq.com/webcourse/index.html#cid=153325&term_id=100175161&taid=743948465362669&type=1024&vid=5285890803719905085)

# 相关术语

  • # 质量方针

    是指“由组织的最高管理者正式发布的该组织的质量宗旨和方向”，体现了该组织（项目）的质量意识和质量追求，是组织内部的行为准则，也体现了顾客的期望和对顾客左侧的承诺。

  • # 质量目标

    是指“在质量方面所追求的目的”，是落实质量方针的具体要求，从属与质量方针，应于利润目标、成本目标、进度目标等相协调。质量目标必须明确、具体，质量用定量化语言进行描述，保证质量目标容易被沟通和理解。分解落实到各部门及项目的全体成员，以便实施、检查和考核。

  • # 全面质量管理 TQM

    是一种全员、全过程、全企业的品质管理，以质量为中心，以全员参与为基础，通过让顾客满意和本组织所有成员及社会收益而达到永续经营的目的。

    4个核心特征：

    • 全员参加的质量管理
    • 全过程的质量管理
    • 全方面方法的质量管理
    • 全面结果的质量管理

# 质量标准

ISO 9000：2000 （等同于国家标准 GB/T 19000-2000）
ISO 9000 族标准可帮助各种类型和规模的组织实施并运行有效的质量管理体系，能伴奏组织增进顾客满意。

  • # ISO 9000

    表述质量管理体系基础知识并规定质量管理体系术语。

  • # ISO 9001

    规定质量管理体系要求，用于组织证实其质量有提供满足顾客要求和适用的法规要求的产品的能力，目的在于增加顾客满意。

  • # ISO 9004

    提供考虑质量体系的有效性和效率两方面的指南。目的在于组织绩效改进和顾客及其他相关方满意。

  • # ISO 19011

    提供审核质量和环境管理体系指南。

# ISO 9000 质量管理的8项原则：

  • 以顾客为中心
    组织依存于其顾客，因此组织应理解顾客当前和未来的需求，满足顾客要求并争取超越顾客期望。
  • 领导作用
    领导者确立本组织统一的宗旨和方向，应创造并保持员工能充分参与实现组织目标的内部环境。
  • 全员参与
    各级人员是组织之本，只有他们的充分参与，才能使他们的才干为组织获益。
  • 过程方法
    将相关的活动和资源作为过程进行管理，可以更高效地得到期望的结果。
  • 管理的系统方法
    识别、理解和管理作为体系的相互关联的过程，有助于组织实现其目标的效率和有效性。
  • 持续改进
    组织总体业绩的持续改进应是组织的一个永恒目标。
  • 基于事实的决策方法
    有效决策时建立在数据和信息分析基础上。
  • 与供方互利的关系
    组织与供方相互依存，互利的关系可增强双方创造价值的能力。

8项原则帮助组织达到持续成功。

# 1.规划质量管理

识别项目及其可交付成果的质量要求和质量，并准备对策确保符合质量要求，为整个项目提供管理和确认质量的指南和方向。
简单来说就是准备对策，确保符合质量要求。

# 输入

  • 项目管理计划
    使用到的信息包括（但不限于）：范围基准(项目范围说明书、WBS、WBS词典)、进度基准、成本基准、其他管理计划等
  • 干系人登记册
  • 风险登记册
  • 需求文件
    记录项目应该满足的、与干系人期望有关的需求。
  • 事业环境因素和组织过程资产

# 技术与工具

  • # 成本收益分析法

    比较每个质量活动可能的成本与预期的效益。
    达到质量要求的主要效益包括：减少返工、提高生产率、降低成本、提升干系人满意度及提升赢利能力。

警告

  • # 质量成本法

    在产品生命周期中发生的所有成本，包括为预防不符合要求、为评价产品或服务是否符合要求，以及因为达到要求而发生的所有成本。

    • 一致性成本

      • 预防成本
        培训、流程文档化、设备、选择正确的做事时间
      • 评价成本
        测试、破坏性测试导致的损失、检查

    • 非一致性成本

      • 内部失败成本
        返工、废品
      • 外部失败成本
        责任、保修、业务流失

  • # 标杆对照 Benchmarking

    将实际或计划的项目实践与可比项目的实践进行对照，以便识别最佳实践，形成改进意见，并为绩效考核提供依据。

  • # 实验设计 DOE

    用来识别哪些因素会对正在生产的产品或正在开发的流程的特定变量产生影响，确定测试的数量和类别，以及这些测试对质量成本的影响。有助于产品或过程的优化，降低产品或性能对各种环境变化或制造过程变化的敏感度。
  • 其他
    头脑风暴、力场分析、名义小组等。

# 输出

  • # 质量管理计划

    描述如何实施组织的质量政策，以及项目管理团队如何达到项目的质量要求。可以是正式的或非正式的，也可以是高度概括或详细的。应在项目早期就对质量管理计划进行评审，确保决策时基于准确信息，更加关注项目的价值定位，降低因返工而造成的成本超支和进度延误。

    公差，是测量指标的可允许变动范围。

  • # 过程改进计划

    详细说明项目管理过程和产品开发过程进行分析的各个步骤，以识别增值活动，需要考虑的方面有：

    • 过程边界
      描述过程的目的、过程的开始和结束、过程的输入输出、过程责任人和干系人。
    • 过程配置
      含有确定界面的过程图形，以便与分析。
    • 过程测量指标
      与控制界限一起，用于分析过程的效率。
    • 绩效改进目标
      用于指导过程改进活动。

  • # 质量测量指标

    描述项目或产品属性，以及控制质量过程将如何对属性进行测量。

  • # 质量核对单

    具体列出各项内容，用来核实所要求的一系列步骤是否已得到执行。应涵盖在范围基准中定义的验收标准。

  • 项目文件更新
    如：干系人登记册、责任分配矩阵、WBS和WBS词典等。

# 2.实施质量保证

审计质量要求和质量控制测量结果，确保采用合理的质量标准和操作性定义，促进质量过程改进。建立对未来输出或未完输出(也称正在进行的工作)将在完成时满足特定的需求和期望的信息。通过规划过程预防缺陷或正在进行的工作检查出缺陷，保证质量的确定性。质量保证部门经常对质量保证活动进行监督，并向相关干系人提供质量保证支持。
简单来说就是促进质量过程改进。

# 输入

  • 质量管理计划
  • 过程改进计划
  • 质量测量指标
  • 质量控制测量结果
  • 项目文件

# 技术与工具

  • # 质量审计

    又称质量保证体系审核，是对具体质量管理活动的结构性的评审。
    质量审计的目标是：

    • 识别全部正在实施的良好及最佳实践
    • 识别全部违规做法、差距及不足
    • 分享所在组织或行业中类似项目的良好实践
    • 积极、主动地提供协助，以改进过程的执行，从而帮助团队提高生产效率
    • 强调每次审计都应对组织经验教训的积累做出贡献

    质量审计可以是事先安排的，也可随机进行。由具体领域中有专长的内部审计师或第三方组织都可以实施质量审计，可由外部或内部审计师进行。

    质量审计还可确认已批准的变更请求(包括更新、纠正措施、缺陷补救和预防措施)的实施情况。

  • # 过程分析

    按照过程改进计划中概括的步骤来识别所需的改进。检查在过程运行期间遇到的问题、制约因素，以及发现的非增值活动。

    过程分析包括：根本原因分析——用于识别问题、探究根本原因，并制定预防措施。

  • # 七种基本质量工具

    七种基本质量工具，用于在PDCA循环的框架内解决与质量相关的问题，分为：老七工具、新七工具。

    警告

    • # 老七工具

      包括：因果图、流程图、核查表、帕累托图、直方图、控制台、散点图。 因流检帕直控散

      • # 因果图

        又称鱼骨图或石川馨图，创始人石川馨。
        是一种发现问题“根本原因”的分析方法。
        问题陈述放在鱼骨头部，作为起点。
        鱼刺上尽可能列出每种原因的可能性。

      • # 流程图

        也称过程图，用来显示在一个或多个输入转化成一个或多个输出的过程中，所需要的步骤顺序和可能分支。
        通过映射SIPOC模型中的水平价值链的过程明细，来显示活动、决策点、分支循环、并行路径及整体处理顺序。
        有助于了解和估算一个过程的质量成本。

      • # 核查表

        又称计数表，用于收集数据的查对清单。
        合理排列各种事项，以有效地手机关于潜在质量问题的有用数据。
        收集的关于缺陷数量或后果的数据，又经常用帕累托图来显示。

      • # 帕累托图 Pareto

        一直特殊的垂直条形图，用于识别造成大多数问题的少数重要原因。
        横轴显示原因类别，作为有效的概率分别，涵盖100%的可能观察结果。
        横轴上每个特定原因的相对频率逐渐减少。

      • # 直方图

        一种特殊形式的条形图，用于描述集中趋势、分散程度和统计分布形状。
        不考虑时间对分布内的变化影响。

      • # 控制图

        根据协议要求而制定规范上限和下限，反映了可允许的最大值和最小值，超出规范界限的可能受处罚。
        用来明确一个过程是否稳定，或者是否具有可预测的绩效。
        一张实时展示项目进展信息的图表。
        可以判断某一过程处于控制之中还是处于失控状态。

        # 七点运行定律

        是指如果在一个质量控制图中，一行上的7个数据点都低于平均值或高于平均值，或者都是上升的，或者都是下降的，那么这个过程就需要因为非随机问题而接受检查。

        如果一个过程处于控制之中，这一过程的所有变量都由随机事件引发，此时的过程不需要调整；
        如果处于失控状态，这一过程产生的变量由非随机事件引发，此时需要确认这些非随机事件的原因，通过调整过程来修改或清楚他们。

        控制图可用于监测各种类型的输出变量，虽然控制图最常用来跟踪批量生产中的重复性活动，但也可用来监测成本与进度偏差、产量、范围变更频率和其他管理工作成果，以便帮助确定项目管理过程是否受控。

      • # 散点图

        可以显示两个变量之间是否有关系，一条斜线上的数据点距离越近，说明两个变量之间的相关性就越密切。
        相关性可能成正相关、负相关或不存在。

    • # 新七工具

      包括：亲和图、过程决策程序图、关联图、树形图、优先矩阵图、活动网络图、矩阵图。亲过关树优网矩

      • # 亲和图

        亲和图和心智图相似。针对某个问题，产出可联成有组织的想法模式的各种创意。使用亲和图确定范围分解的结构，有助于WBS 工作分解结构的制定。

      • # 过程决策程序图 PDPC

        用于理解一个目标与达成此目标的步骤之间的关系。有助于制定应急计划，因为它能帮助团队预测那些可能破坏目标实现的中间环节。

      • # 关联图

        关系图的变种，有助于在包含相互交叉逻辑关系(可有多达50个相关项)的中等复杂情形中创新性地解决问题。
        可以使用其他工具(如：亲和图、树形图、鱼骨图等)产生的数据来绘制关联图。

      • # 树形图

        也称系统图，可用于表现诸如：WBS 工作分解结构、RBS(风险分解结构)、OBS(组织分解结构)的层次分解结构。
        用层次分解形式直观地展示父子关系。
        图形可以是横向(如RBS)或纵向(如OBS)。

      • # 优先矩阵图

        用来识别关键事项和合适的备选方案，通过一系列的决策，排列出备选方案的优先顺序。

      • # 网络活动图

        过去称为箭头图，包括两种形式：AOA 活动箭线图(双代号图)、AON 活动节点图(单代号图)。

        活动网络图同项目进度计划编制方法一起使用，如：计划评审技术 PERT、关键路径法 CPM、紧前关系绘图法 PDM。

      • # 矩阵图

        使用矩阵结构对数据进行分析，在行列交叉的位置展示因素、原因和目标之间的关系强弱。

  • # 统计抽样

    从目标总体中抽取一部分相关样本用于检查和测量，以满足质量管理计划中的规定。
    抽样的频率和规模应在质量规划管理过程中确定，以便在质量成本中考虑测试数量和预期废料等。

    一般信息系统用不上！！！生产、制造业常用。

  • # 检查

    检验工作产品，以确定是否符合书面标准。

    笔记

    检查也可称为：审查、同行审查、审计、巡检。

    可确认缺陷补救。

  • 变更请求
  • 项目文件更新
    各种管理计划随之更新，并更新相关的：质量审计报告、培训计划、过程文档等。
  • 组织过程资产更新
    例如：组织的质量标准、质量管理系统等。

# 3.控制质量

监督并记录质量活动执行结果，以便评估绩效，并推荐必要的变更。识别过程低效或产品质量低劣的原因，建议并采取相应措施消除这些原因，同时确认项目的可交付陈国及工作满足主要干系人的既定需求，足以进行最终验收。

确认范围与控制质量的区别

# 输入

  • 项目管理计划
    包含质量管理计划，描述如何开展质量控制。
  • 质量测量指标
    描述项目或产品属性及其测量方式。

    • # 平均故障间隔时间 MTBF

    • # 平均修复时间 MTTR

  • 质量核对单
  • 工作绩效数据
    包括：
    • 实际技术性能(与计划比较)

    • 实际成本绩效(与计划比较)

  • 可交付成果

    如：问题与缺陷报告程序及沟通政策、组织的质量标准和政策、标准化的工作指南等。
